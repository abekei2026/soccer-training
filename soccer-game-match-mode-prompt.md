# 「しあいモード(7人制・フルコート)」追加開発依頼

既存のサッカー戦略学習ゲーム「サッカーあたまトレーニング」に、新しいゲームモードを追加してください。既存の3vs3モードは変更せずそのまま残し、モード選択で切り替えられるようにします。

---

## 1. モード構成

- タイトル画面にモード選択を追加:
  - **きほんモード**: 既存の3vs3+GK(現状のまま)
  - **しあいモード**: 今回追加する7人制・フルコート
- 共通の操作方法・スローモーション・パスコース可視化などの既存機能は、しあいモードでもそのまま使う

## 2. しあいモードの基本構成

- **7人制**: 各チーム GK 1人 + フィールドプレーヤー6人
- **フルコート**(両側にゴール)。コートは既存モードよりだいぶ大きくする
- 攻撃側(青)はプレイヤー1人+味方AI 5人+GK。守備側(赤)はAI 6人+GK
- 1試合は前後半なしの1本(時間は設定で選択: 2分 / 3分 / 5分)
- 得点が入ったらセンターサークルからキックオフで再開

## 3. カメラとミニマップ(必須)

フルコートは1画面に収まらないため、以下を実装する:

- **ボール追従カメラ**: ボール(またはボール保持者)を中心に画面がスクロール。急な動きで酔わないよう、カメラの追従はなめらかに補間する
- **ミニマップ**: 画面隅にコート全体の俯瞰図を常時表示。全選手とボールの位置を点で表示。自分の選手は目立つ色にする
- ズームレベルは「自分の周囲+パスが届く範囲の味方が見える」程度を基準に調整する

## 4. 守備側のゾーンディフェンスシステム(このモードの核)

### 4-1. エリア分割

- 守備側のハーフコートを **6エリア** に分割する:
  - 縦(コート幅方向)に3分割: 左 / 中央 / 右
  - 横(ゴール方向)に2分割: 前(ハーフウェーライン側)/ 後ろ(自ゴール側)
- フィールドプレーヤー6人それぞれに1エリアをホームエリアとして割り当てる
- 各DFはボールが遠いとき、自分のホームエリア内を基本位置とする

### 4-2. 守備の基本原則

各DFは毎フレーム以下の優先順位で行動を決める:

1. **ボールが自分の集結半径内にある場合**(4-3参照):
   - ボール保持者に最も近いDF 1人がプレスに行く(チャレンジ役)
   - それ以外の半径内DFは、保持者から近い攻撃選手へのパスコースを切る位置に立つ(カバー役)
2. **ボールが集結半径の外にある場合**:
   - 自分のホームエリア内(または最も近く)にいるボールを持っていない攻撃選手をマークする
   - マークは常に**ゴールサイド**(攻撃選手と自ゴールを結ぶ線上、攻撃選手寄り)に立つ
   - エリア内に攻撃選手がいなければホームエリアの基準位置に戻る

### 4-3. 集結半径(設定項目)

- 「エリアの1辺」を長さの単位1とする。**単位はエリアの横幅(コート幅÷3)を基準**とし、コード内に明記する
- 設定画面で集結半径を選択できるようにする: **1 / 1.5 / 2**
- 半径はDFのホームエリアの中心からボールまでの距離で判定する
- 半径が大きいほど多くのDFがボールに集まり(コンパクトな守備)、小さいほど各自がエリアを守る(広がった守備)
- 設定はしあいモード開始前の画面で選択。試合中も一時停止メニューから変更可能にする

### 4-4. ボールサイドへのスライド

- ボールが左サイドにあるとき、守備ブロック全体が左に寄る(実際のゾーンディフェンスの「スライド」)
- 実装: 各DFの基準位置を、ホームエリア中心からボール方向へエリア幅の0.3〜0.5倍だけオフセットする。エリアの割り当て自体は変えない

### 4-5. 学習用の可視化(トグル)

- **エリア表示ON/OFF**: 6エリアの区切り線と、各DFのホームエリアを薄い半透明色で表示
- **集結半径表示ON/OFF**: ボールを中心に現在の集結半径の円を薄く表示し、半径内で反応しているDFを軽くハイライト
- 既存のパスコース可視化(緑線/赤点線)もこのモードで機能させる

## 5. 攻撃側AI(味方5人)の役割分担

団子サッカーにならないよう、簡易フォーメーションを持たせる:

- 役割: FW 2人 / MF 2人 / DF 2人(うち1人をプレイヤーが操作。操作対象はMFの1人を推奨)
- 各役割に基準ポジション(コート上の担当範囲)を持たせ、ボール位置に応じて範囲内で動く
- ボール保持時のパス判断ロジックは既存モードと同じ(コースが通っていてフリーな味方を優先、前進パスを好む)
- FWは裏のスペースへの走り込みをときどき行う

## 6. ルールの簡略化

- **オフサイドは実装しない**(将来の拡張とする)
- ボールがタッチライン・ゴールラインから出たら、スローインやコーナーキックは行わず、**出た地点の近くから相手ボールでリスタート**(2秒程度のフリーズ後に再開)
- ファウルなし(既存モード同様、接触は自動処理)

## 7. スコアと学習支援

- 得点設計は既存モードを流用。ただしフルコートなので「前進パス」の判定はコートの縦方向で行う
- ラウンド終了画面のコーチアドバイスに、しあいモード用を追加:
  - 例:「あいてが 少ないほうの サイドに ボールを うごかそう!」(逆サイド展開のヒント)
  - 例:「DFが あつまってきたら フリーの みかたが いるよ」
- 文言は既存モード同様、小学生向けにひらがな・カタカナ多め、前向きな表現で

## 8. 設定画面(しあいモード開始前)

以下を選択できるようにする:

| 項目 | 選択肢 | 初期値 |
|---|---|---|
| 集結半径 | 1 / 1.5 / 2 | 1.5 |
| 試合時間 | 2分 / 3分 / 5分 | 3分 |
| エリア表示 | ON / OFF | ON |
| 集結半径表示 | ON / OFF | OFF |
| 敵の強さ | よわい / ふつう / つよい | ふつう |

「敵の強さ」はDFの移動速度と反応速度(判断の更新間隔)で差をつける。

## 9. パフォーマンス要件

- 選手が合計14人+ボールになるため、60fpsを維持できるよう毎フレームの距離計算を効率化する(必要なら判断ロジックの更新は数フレームおきにする)
- スマホ実機(iPhone/Android)で快適に動くことを優先する

## 10. 実装フェーズ

**まずPhase 1のみ実装して止めてください。** 各Phase完了時に動作確認ポイントを教えてください。

- **Phase 1**: モード選択画面、フルコート描画、ボール追従カメラ+ミニマップ、攻撃側7人(フォーメーション付き)のパス回し。守備側はまだGKのみ
- **Phase 2**: 守備側6人のゾーンディフェンス(エリア割り当て、集結半径ロジック、マーク、スライド)
- **Phase 3**: 設定画面(集結半径・試合時間・敵の強さ)、エリア/半径の可視化トグル
- **Phase 4**: 得点・コーチアドバイスのしあいモード対応、アウトオブバウンズ処理、パフォーマンス調整
